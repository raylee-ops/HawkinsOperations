# Code of Conduct

## Our Commitment

This repository is a professional security operations portfolio. We are committed to maintaining a respectful, collaborative, and harassment-free environment for everyone who contributes or engages with this project.

## Our Standards

### Expected Behavior

**Positive contributions include:**
- Providing constructive feedback on detection rules, playbooks, or documentation
- Suggesting improvements to detection logic or incident response procedures
- Reporting security issues responsibly (see [SECURITY.md](SECURITY.md))
- Helping improve documentation clarity
- Sharing knowledge and expertise in security operations
- Being respectful of differing viewpoints and experiences

### Unacceptable Behavior

**Examples of unacceptable behavior include:**
- Hostile, offensive, or derogatory comments
- Sharing real credentials, API keys, or sensitive information in issues/PRs
- Proposing detection rules that target specific individuals or organizations
- Attempting to introduce backdoors or malicious code
- Publishing others' private information without permission
- Any conduct that would be inappropriate in a professional security environment

## Scope

This Code of Conduct applies to all project spaces, including:
- GitHub repository (issues, pull requests, discussions)
- Code review comments
- Documentation contributions
- Any communication related to this project

## Security-Specific Guidelines

Given this is a security operations portfolio:

1. **Responsible Disclosure:** Report security vulnerabilities privately via [SECURITY.md](SECURITY.md)
2. **No Weaponization:** Detection rules should focus on defensive use cases
3. **Sanitization Required:** All contributions must follow [PROOF_PACK/EVIDENCE_CHECKLIST.md](PROOF_PACK/EVIDENCE_CHECKLIST.md)
4. **No Real Data:** Never include real IPs, hostnames, credentials, or PII

## Enforcement

### Reporting

If you experience or witness unacceptable behavior, please report it by:
- Creating a private security advisory on GitHub
- Contacting the repository owner via the email in commit history

### Consequences

Violations may result in:
- Warning with request to correct behavior
- Temporary or permanent ban from contributing
- Removal of contributed content
- Report to GitHub for Terms of Service violations

### Attribution

This Code of Conduct is adapted from the [Contributor Covenant](https://www.contributor-covenant.org/), version 2.1, tailored for a security operations portfolio context.
